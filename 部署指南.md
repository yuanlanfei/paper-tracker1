# 🚀 GitHub Pages 部署指南

## 📋 部署流程（5分钟搞定）

### 第一步：创建 GitHub 仓库

1. 访问 https://github.com/new
2. 仓库名填写：`pa-paper-tracker`（或其他名字）
3. 选择 **Public**（公开）
4. 点击 **Create repository**

### 第二步：上传代码

#### 方式A：直接上传文件（最简单）

1. 在新仓库页面，点击 **uploading an existing file**
2. 将 `f:\文献爬取\github-pages\` 下的所有文件拖拽上传：
   - `.github/workflows/crawl.yml`
   - `data/papers.json`
   - `data/data.json`
   - `scripts/crawler.py`
   - `scripts/export.py`
   - `index.html`
   - `README.md`
3. 点击 **Commit changes**

#### 方式B：使用 Git 命令

```bash
cd f:\文献爬取\github-pages
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/你的用户名/仓库名.git
git push -u origin main
```

### 第三步：配置 Secrets

1. 进入仓库 → **Settings** → **Secrets and variables** → **Actions**
2. 点击 **New repository secret**
3. 填写：
   - **Name**: `EMAIL`
   - **Value**: 你的邮箱地址
4. 点击 **Add secret**

### 第四步：启用 GitHub Pages

1. **Settings** → **Pages**
2. **Build and deployment** → **Source**
3. 选择 **GitHub Actions**
4. 保存

### 第五步：手动触发首次爬取

1. 点击仓库顶部的 **Actions** 标签
2. 点击左侧的 **"每日爬取SSCI文献"**
3. 点击右侧的 **Run workflow** 按钮
4. **Branch**: 选择 `main`
5. **mode**: 选择 `initial`（爬取最近30天）
6. 点击 **Run workflow**
7. 等待 5-10 分钟完成

### 第六步：访问网站

爬取完成后，访问：
```
https://你的用户名.github.io/仓库名/
```

或者查看 **Settings** → **Pages** 中的链接。

---

## ✅ 验证部署成功

1. 进入 **Actions** 页面，确认工作流显示绿色 ✓
2. 查看 **data/data.json** 文件，确认有数据
3. 访问网站链接，确认能正常显示

## 🐛 常见问题

### Q: 页面显示 "数据加载失败"？
- 确认 Actions 工作流已成功运行
- 确认 data/data.json 文件已生成
- 等待几分钟再刷新（GitHub Pages 有缓存）

### Q: 爬虫运行失败？
- 检查是否正确设置了 `EMAIL` secret
- 查看 Actions 日志排查错误

### Q: 如何修改爬取时间？
- 编辑 `.github/workflows/crawl.yml`
- 修改 `cron: '0 0 * * *'` 为你想要的时间

### Q: 访问速度慢？
- 国内访问可能较慢，可以使用代理
- 或考虑部署到 Vercel/Netlify（步骤类似）
